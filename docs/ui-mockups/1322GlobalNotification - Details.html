Description

Provides global, non-blocking feedback to the user, including banners for persistent states (like being offline) and toasts for temporary notifications (like success, error, or sync status).

Design System Alignment

Utilizes the design system's semantic color palette, spacing scale, and typography for consistent feedback presentation.

Responsive Specifications

Mobile Adaptations

  • The offline banner is full-width at the top of the viewport.
  • Toasts appear near the bottom, are full-width with some margin, and stack vertically.

Tablet Adaptations

  • No specific changes from mobile; behavior remains consistent.

Desktop Enhancements

  • Toasts have a `max-width` and are horizontally centered at the bottom of the viewport, appearing more compact.

Accessibility Features

Keyboard Support

The close button on toasts is focusable and can be activated with Enter/Space.

Screen Reader Support

Components use `role="status"` to be identified as live regions. `aria-live="polite"` is used for the persistent banner, while `aria-live="assertive"` is used for time-sensitive toasts to ensure they are announced promptly.

Focus Management

The close button has a clear focus indicator. Opening a notification does not trap focus.

Aria Attributes

  • role="status"
  • aria-live="polite"
  • aria-live="assertive"
  • aria-label on close buttons

Color Contrast

All text/background color combinations for variants meet WCAG 2.1 AA requirements.
