Description

A modal dialog that interrupts the user's workflow to confirm a critical action. It requires a user response before they can continue. It is used for actions like confirming a deletion (destructive) or saving changes (informational).

Design System Alignment

Uses design system tokens for colors (primary, danger, neutrals), typography, spacing, border-radius, and shadows. It is composed of atomic Button and Icon components.

Responsive Specifications

Mobile Adaptations

  • Modal panel is full-width.
  • Modal appears from the bottom of the screen.
  • Panel corners are squared off at the bottom edge of the viewport.

Tablet Adaptations

  • Switches to desktop view: centered panel with max-width.

Desktop Enhancements

  • Centered modal with a backdrop overlay.
  • Scale-in animation on open for a subtle effect.

Accessibility Features

Keyboard Support

Escape key closes the modal. Tab key cycles focus within the modal. Focus is trapped inside.

Screen Reader Support

Uses `role='dialog'`, `aria-modal='true'`, `aria-labelledby`, and `aria-describedby` for a proper dialog experience. Close button has `aria-label`.

Focus Management

Focus is programmatically moved to the modal panel when opened, and returned to the trigger element when closed. Focus is trapped inside while open.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby='modal-title-ID'
  • aria-describedby='modal-desc-ID'

Color Contrast

All text and button colors meet WCAG 2.1 AA contrast ratios.
