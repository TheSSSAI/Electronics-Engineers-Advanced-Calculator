Description

A modal overlay component that hosts the application's help system. It includes states for loading content, displaying content, and handling errors. It is designed to be fully responsive, appearing as a large modal on desktops and a full-screen view on mobile devices.

Design System Alignment

This component utilizes the design system's color palette, spacing, typography, and shadow tokens. It is composed of atomic Icon and Spinner components.

Responsive Specifications

Mobile Adaptations

  • The component occupies the full screen (100% width and height).
  • Panel has no border-radius or shadow, appearing as a native full-screen view.

Tablet Adaptations

  • At 768px and above, the component transitions to the desktop modal view.

Desktop Enhancements

  • The component is displayed as a large modal with a backdrop overlay.
  • The panel has a max-width and max-height, border-radius, and a prominent shadow to give it depth.
  • A scale-in animation is applied on open for a smoother user experience.

Accessibility Features

Keyboard Support

The close button is focusable and can be activated with Enter/Space. The Escape key should be programmatically wired to close the modal. Focus must be trapped within the modal when open.

Screen Reader Support

The panel has `role='dialog'`, `aria-modal='true'`, and `aria-labelledby` to announce its purpose. The close button has an explicit `aria-label`.

Focus Management

Focus should be moved to the panel on open and returned to the trigger element on close. Focus must be trapped within the panel's interactive elements while open. All interactive elements have a clear `:focus-visible` state.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby='help-system-title'
  • aria-label='Close help'

Color Contrast

All text and UI elements meet WCAG 2.1 AA color contrast requirements using the design system tokens.
