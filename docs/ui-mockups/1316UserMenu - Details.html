Description

Provides a dropdown menu for user-related actions. It displays a login/register button for logged-out users and an avatar with a menu for authenticated users, offering options like profile management and logout.

Design System Alignment

This component utilizes the design system's color palette for backgrounds and text, spacing for padding, typography for font styles, and border-radius for shaping. It depends on the atomic Button and Icon components for its composition.

Responsive Specifications

Mobile Adaptations

  • Dropdown positioning with `right: 0` ensures it doesn't overflow the viewport on the right.
  • For very small screens, a media query could be added to make the dropdown full-width (`width: calc(100vw - 32px); left: 16px; right: 16px;`).
  • All interactive elements (trigger, menu items) have a minimum height of 44px to ensure adequate touch targets.

Tablet Adaptations

  • The component maintains its desktop appearance and behavior.

Desktop Enhancements

  • The component uses hover and focus states for clear interactivity.
  • The shadow on the dropdown provides a sense of depth and separation from the main content.

Accessibility Features

Keyboard Support

The trigger button is focusable. When the dropdown is open, Tab key navigates through the menu items. Escape key (if JS is used) would close the menu.

Screen Reader Support

The trigger button uses `aria-haspopup='true'` and `aria-expanded` to announce its function. The dropdown has `role='menu'` and items have `role='menuitem'` when implemented with JavaScript.

Focus Management

A visible focus ring is applied to the trigger and all menu items. The showcase uses `:focus-within` to demonstrate how focus should be contained within the component when open.

Aria Attributes

  • aria-haspopup
  • aria-expanded
  • aria-controls

Color Contrast

All text and background color combinations adhere to WCAG 2.1 AA standards by using the predefined design system tokens.
