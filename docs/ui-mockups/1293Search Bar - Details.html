Description

A responsive search bar component that functions as an expanded input on desktop and a modal-triggering icon on mobile. It includes states for loading, results, and no results.

Design System Alignment

Utilizes the design system's primary and neutral color palettes, spacing system, typography, and form input specifications. Focus states and shadows are consistent with the established interaction patterns.

Responsive Specifications

Mobile Adaptations

  • The search bar collapses into a single icon button.
  • Tapping the icon opens a full-screen modal/overlay.
  • The search input and results are displayed within the overlay, optimizing for vertical space.

Tablet Adaptations

  • The component switches to the expanded desktop view at the 768px breakpoint.

Desktop Enhancements

  • The search bar is persistently expanded.
  • Search results are displayed in a dropdown menu directly below the input field.
  • Hover states are active for search result items.

Accessibility Features

Keyboard Support

The search input is focusable with Tab. Results can be navigated with arrow keys (JS required) or Tab. Mobile trigger is focusable and activatable with Enter/Space.

Screen Reader Support

The input has a visually hidden label. The search results dropdown uses ARIA attributes like `aria-controls` and `aria-expanded` (managed via JS). The mobile trigger has an `aria-label`.

Focus Management

A clear, visible focus indicator is applied to the input and interactive results. In the mobile overlay, focus should be trapped (requires JS).

Aria Attributes

  • role="search"
  • aria-label="Search documentation"
  • aria-controls
  • aria-expanded

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
