Description

A responsive navigation sidebar for the documentation section of the application. It is persistently visible on desktop viewports and functions as an accessible, off-canvas drawer on mobile viewports.

Design System Alignment

This organism composes the 'Sidebar Navigation Item' molecular component and utilizes the design system's tokens for background colors, spacing, typography, shadows, and responsive breakpoints to ensure a consistent user experience.

Responsive Specifications

Mobile Adaptations

  • Sidebar is hidden off-canvas by default.
  • A hamburger menu (not part of this component, but assumed to exist in the header) toggles visibility.
  • An overlay/backdrop is displayed to obscure the main content when the sidebar is open.
  • Sidebar slides in from the left to cover part of the screen.

Tablet Adaptations

  • At the 768px breakpoint, the sidebar transitions to the desktop-persistent layout.

Desktop Enhancements

  • Sidebar is persistently visible on the left side of the screen.
  • It occupies a fixed width, pushing the main content to the right.
  • The sidebar is 'sticky', remaining in place as the user scrolls the main content.

Accessibility Features

Keyboard Support

The mobile drawer can be closed with the Escape key. All internal navigation items are focusable and activatable via keyboard.

Screen Reader Support

On mobile, the drawer has `role='dialog'` and `aria-modal='true'` to inform screen reader users it is an overlay. A visually hidden close button should be included for screen reader navigation.

Focus Management

When the mobile drawer opens, focus is programmatically moved to the first item inside. Focus is trapped within the drawer until it is closed.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby='sidebar-title'

Color Contrast

All text and interactive elements within the sidebar meet WCAG 2.1 AA contrast requirements.
