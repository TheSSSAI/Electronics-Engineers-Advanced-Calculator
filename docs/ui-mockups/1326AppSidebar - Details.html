Description

The main application navigation component. Functions as a persistent, visible sidebar on larger viewports and collapses into a modal navigation drawer on smaller viewports.

Design System Alignment

This component uses the design system's color palette for backgrounds and active states, the spacing system for layout and item padding, and the typography scale for link text.

Responsive Specifications

Mobile Adaptations

  • Sidebar is hidden off-screen by default.
  • Appears as a modal drawer that slides in from the left.
  • An overlay covers the main content area when the drawer is open.
  • Includes a close button within the sidebar header.
  • Scrolls vertically if navigation items overflow the viewport height.

Tablet Adaptations

  • At the 768px breakpoint, the sidebar transitions to the desktop variant.

Desktop Enhancements

  • Sidebar is persistently visible on the left side of the screen.
  • Occupies a fixed width, pushing the main content to the side.
  • The overlay and mobile close button are hidden.

Accessibility Features

Keyboard Support

All navigation links and the close button are focusable and activatable. When implemented with JavaScript, 'Escape' should close the mobile drawer.

Screen Reader Support

Uses semantic `

Focus Management

Visible `:focus-visible` styles are applied to all interactive elements. For the mobile drawer, JavaScript is required to trap focus within the sidebar when it is open and return focus to the trigger button when closed.

Aria Attributes

  • aria-label for main nav and close button
  • aria-hidden to manage visibility for screen readers
  • JavaScript would add role='dialog' and aria-modal='true' to the mobile drawer for proper semantics.

Color Contrast

All text and background color combinations meet WCAG 2.1 AA contrast requirements.
