Atomic Utility Components

A showcase of non-visual, logic-based atomic components. These utilities form the foundation of the application's data manipulation and developer experience.

A utility to parse strings containing SI prefixes (k, M, m, μ, n, p) into their raw numerical equivalents. It handles valid and invalid inputs gracefully.

Logger

A developer utility for structured, consistent logging. It's a non-visual component that outputs to the browser's developer console.

Usage Example

The Logger provides different levels (info, warn, error) and enriches log messages with context like timestamps and component names. It can be enabled or disabled globally for performance.

// Example of how the Logger might be used in code
import { Logger } from '@/utils/logger';

const log = new Logger('CalculatorComponent');

function handleCalculation() {
  try {
    log.info('Starting calculation...', { expression: '2+2' });
    // ... calculation logic ...
    log.info('Calculation successful.');
  } catch (error) {
    log.error('Calculation failed!', { error });
  }
}
