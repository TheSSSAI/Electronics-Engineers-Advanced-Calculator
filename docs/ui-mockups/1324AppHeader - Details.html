Description

The main application header (AppBar) that is persistent across all views. It contains the main navigation toggle, the application title, the primary mode switcher, and user-specific actions.

Design System Alignment

This component uses design system tokens for background color, text color, spacing, typography, and shadows. It serves as a container for molecular components like ModeSwitcher and UserMenu.

Responsive Specifications

Mobile Adaptations

  • A hamburger icon (`.app-header__nav-toggle`) is displayed on the left to toggle the main navigation sidebar.
  • The application brand/title is hidden to save space.
  • The ModeSwitcher is centered.
  • Utility icons (like Help) may be hidden on the smallest viewports (< 480px) to prevent clutter.

Tablet Adaptations

  • The layout remains the same as mobile, with the hamburger menu still present as the primary navigation trigger.

Desktop Enhancements

  • The hamburger menu icon is hidden (`display: none`).
  • The application brand/title (`.app-header__brand`) is displayed on the left.
  • All utility icons are visible.

Accessibility Features

Keyboard Support

All interactive elements (navigation toggle, utility buttons, ModeSwitcher, UserMenu) are focusable and activatable using Tab, Enter, and Space keys.

Screen Reader Support

The navigation toggle button has an `aria-label` ('Open navigation menu') and `aria-expanded` to announce its state. Other icon buttons also have descriptive `aria-label`s.

Focus Management

All interactive elements have a clear, high-contrast `:focus-visible` state as defined in the design system.

Aria Attributes

  • aria-label
  • aria-expanded
  • aria-controls
  • aria-hidden

Color Contrast

Text and icon colors meet WCAG 2.1 AA contrast ratios against the header background.
