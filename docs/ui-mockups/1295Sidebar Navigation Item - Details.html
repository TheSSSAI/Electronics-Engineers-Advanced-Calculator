Description

A navigation element for the documentation sidebar, supporting both direct document links and collapsible categories for nested navigation. It is designed to be interactive, stateful, and accessible.

Design System Alignment

This component utilizes the design system's color palette for states (primary for active, neutrals for default/hover), typography for legibility, and spacing tokens for consistent padding and touch targets.

Responsive Specifications

Mobile Adaptations

  • Ensures touch targets are sufficiently large due to padding (`var(--space-sm)` top/bottom).
  • Maintains legibility with `var(--font-size-caption)`.

Tablet Adaptations

  • No specific adaptations needed; component scales with the sidebar container.

Desktop Enhancements

  • Includes hover states for discoverability.
  • Visible focus states for keyboard navigation.

Accessibility Features

Keyboard Support

All items are focusable with Tab. Links activated with Enter. Category toggles activated with Enter/Space.

Screen Reader Support

Uses semantic `` for links and `

Focus Management

Provides a clear, visible focus indicator on all interactive elements.

Aria Attributes

  • aria-expanded="true/false" on category toggles.
  • aria-controls="[id]" on category toggles, pointing to the submenu's ID.

Color Contrast

All text and state colors are checked to meet WCAG 2.1 AA contrast ratios.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, and Edge.

Device Testing

Verify tappability and legibility on a mobile viewport.

Accessibility Testing

  • Full keyboard navigation test (Tab, Shift+Tab, Enter, Space).
  • Screen reader test with NVDA or VoiceOver, focusing on the announcement of expanded/collapsed states and active links.
  • Color contrast validation for all text and states.

Performance Testing

Verify smooth animations for collapse/expand transitions.
