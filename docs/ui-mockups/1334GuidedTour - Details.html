Description

An interactive overlay component that guides users through a series of UI elements step-by-step. It consists of a backdrop, a highlighted area focusing on a target element, and a popover with descriptive text and navigation controls.

Design System Alignment

This component uses design system tokens for colors (popover background, text, buttons), shadows, border-radius, typography, and spacing. It is composed of atomic Tooltip and Button components.

Responsive Specifications

Mobile Adaptations

  • The popover has a max-width to prevent it from touching screen edges.
  • Popover positioning logic must account for the viewport to avoid being rendered off-screen.

Tablet Adaptations

  • No specific changes required; the component is flexible by design.

Desktop Enhancements

  • The popover can be positioned to the left, right, top, or bottom of the target element, depending on available space.

Accessibility Features

Keyboard Support

Navigation buttons (Back, Next, Skip) are focusable and can be activated with Enter/Space.

Screen Reader Support

The popover uses `role='dialog'`, `aria-modal='true'`, `aria-labelledby`, and `aria-describedby` to provide a proper dialog experience for screen reader users.

Focus Management

In a live implementation, JavaScript would be required to trap focus within the popover's interactive elements and manage focus when moving between steps.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and interactive elements within the popover meet WCAG 2.1 AA contrast requirements.
