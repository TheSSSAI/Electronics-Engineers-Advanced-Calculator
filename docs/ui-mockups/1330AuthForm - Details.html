Description

An organism component that provides a complete form for user authentication, with variants for both Login and Registration. It handles default, loading, and error states.

Design System Alignment

This component is composed of molecular (FormGroup) and atomic (Button, Checkbox, Link) components. It heavily relies on the design system's tokens for color, typography, spacing, and semantic states (error).

Responsive Specifications

Mobile Adaptations

  • The single-column layout is inherently mobile-friendly.
  • A max-width ensures the form does not stretch unreadably on landscape mobile views.
  • Touch target sizes for buttons and inputs are a minimum of 48px.

Tablet Adaptations

  • The layout remains a single, centered column for optimal focus and readability.

Desktop Enhancements

  • The form is centered within the viewport with a max-width, providing a clean and focused user experience.

Accessibility Features

Keyboard Support

The form is fully navigable using the Tab key. Interactive elements (inputs, links, buttons) are focusable and operable with Enter/Space.
