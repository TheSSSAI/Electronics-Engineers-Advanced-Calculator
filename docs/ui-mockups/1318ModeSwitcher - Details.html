Description

A segmented control component used to switch between a set of related views or modes, such as the main calculator modes (Scientific, Ohm's Law) or angle modes (DEG, RAD, GRAD). It provides clear visual feedback of the currently active selection.

Design System Alignment

This component utilizes the design system's color palette for backgrounds and text, the spacing system for padding, the typography scale for font styles, and border-radius tokens for its shape. It inherits focus-visible styles for accessibility.

Responsive Specifications

Mobile Adaptations

  • The component will wrap its items if they do not fit on a single line.
  • Button padding is slightly reduced, but a minimum touch target size is maintained.
  • The spec suggests collapsing to a dropdown, but for a static HTML mockup, wrapping is a more feasible demonstration of responsiveness.

Tablet Adaptations

  • Maintains the desktop appearance of a segmented control bar.

Desktop Enhancements

  • Displays as a horizontal segmented control bar.
  • Includes hover effects for non-active buttons.

Screen Reader Support

The container group has an `aria-label` describing its purpose (e.g., 'Main Calculator Modes'). The active button uses `aria-pressed='true'` to programmatically announce the current selection.

Focus Management

A clear, high-contrast focus ring is applied using `:focus-visible` to avoid cluttering the UI for mouse users while providing essential feedback for keyboard users.

Aria Attributes

  • role='group'
  • aria-label
  • aria-pressed='true'/'false'

Color Contrast

All text and background color combinations for default, active, and hover states meet WCAG 2.1 AA contrast requirements.
