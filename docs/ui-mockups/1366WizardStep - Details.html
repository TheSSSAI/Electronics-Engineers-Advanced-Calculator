Description

A layout component that represents a single step within a multi-step wizard process. It visually indicates the state of the step (active, inactive, complete) and contains the content and navigation for that specific step.

Design System Alignment

This component uses the design system's primary and neutral color palette for state indication, spacing tokens for layout, and typography for step labels. It also depends on the 'Button' atomic component for its navigation actions.

Responsive Specifications

Mobile Adaptations

  • Step labels are hidden to save horizontal space, leaving only the numbered indicators.
  • The content area for the active step is given a max-height and overflow-y: auto to ensure long content is scrollable and doesn't break the layout.
  • Padding within the content wrapper is reduced.

Tablet Adaptations

  • The component maintains its desktop appearance, with full step labels visible.

Desktop Enhancements

  • Full visibility of step labels and indicators.
  • Hover effects on completed steps to indicate they are clickable.

Accessibility Features

Keyboard Support

Completed steps are focusable via Tab. Navigation buttons ('Back', 'Next') are focusable and activatable via Enter/Space.

Screen Reader Support

The stepper is wrapped in an `

` with an `aria-label`. The current step has `aria-current='step'`. Each step indicator has a visually hidden label for context on mobile (e.g., 'Step 1, Basic Information, Complete').

Focus Management

When a new step becomes active, focus should be programmatically moved to the first focusable element within that step's content (e.g., a heading or input field).

Aria Attributes

  • aria-label on the parent
  • aria-current='step' on the active step's link/button

Color Contrast

All text and state indicators meet WCAG 2.1 AA contrast requirements against their backgrounds.
